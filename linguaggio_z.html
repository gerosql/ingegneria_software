Cos'è il Linguaggio Z?

Il linguaggio Z, sviluppato all’Università di Oxford, è un formalismo per la specifica rigorosa dei sistemi software, basato sulla logica del primo ordine e la teoria degli insiemi. Utilizza schemi per descrivere aspetti statici e dinamici.

Costrutti di Base

  • Insiemi e Funzioni: Utilizza concetti della teoria degli insiemi e logica.
  • Stato: Variabili raggruppate in schemi di stato.
  • Invarianti di Stato: Condizioni valide in ogni possibile stato.

Esempio di Schema di Stato: Agenda

Agenda
  conosciuto: ℙ NOME
  compleanno: NOME ⇸ DATA
―――――――――――――――――――――――
  dom compleanno = conosciuto

Invariante: Ogni nome in compleanno appartiene a conosciuto.

Operazioni Principali

Inserimento di un compleanno

Aggiungi
  ΔAgenda
  nome?: NOME
  data?: DATA
―――――――――――――――――――――――
  nome? ∉ conosciuto
  conosciuto′ = conosciuto ∪ {nome?}
  compleanno′ = compleanno ∪ {nome? ↦ data?}

Ricerca di un compleanno

Ricerca
  ΞAgenda
  nome?: NOME
  data!: DATA
―――――――――――――――――――――――
  nome? ∈ conosciuto
  data! = compleanno nome?

Promemoria per una data

Promemoria
  ΞAgenda
  data?: DATA
  nomi!: ℙ NOME
―――――――――――――――――――――――
  nomi! = { n: NOME | compleanno n = data? }

Casi Particolari e Composizione

Per gestire casi particolari, si definisce un tipo di risultato:

RISULTATO ::= ok | già_presente | non_presente

E si usano schemi ausiliari combinati tramite operatori come union e rinominazione.

AggiungiRobusto = SpecPart ⨄ Aggiungi

Perché usare il Linguaggio Z?

  • Precisione: Definizione formale delle operazioni.
  • Verificabilità: Strumenti automatici per controllo e validazione.
  • Modularità: Schemi riutilizzabili e composizione semplificata.
