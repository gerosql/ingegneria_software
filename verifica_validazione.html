Verifica e Validazione

Introduzione

Nell'ingegneria del software, verifica e validazione sono due attività cruciali svolte per garantire la qualità, l'affidabilità e la conformità del software rispetto alle aspettative degli utenti.

1. Verifica

La verifica è il processo di valutazione di un sistema o componente software per determinare se soddisfa i requisiti specificati durante la fase di sviluppo. Si verifica che il software sia stato sviluppato correttamente e rispetti gli standard richiesti.

  • Revisioni del codice sorgente
  • Test statici come l'analisi statica del codice
  • Ispezione dei documenti per individuare errori e difetti

2. Validazione o Convalida

La validazione è il processo di conferma che il software soddisfi effettivamente i requisiti del cliente e sia adatto all'uso previsto. Si verifica se il software risolve efficacemente il problema per cui è stato progettato.

  • Test dinamici come test funzionali
  • Test di accettazione
  • Test dell'usabilità per assicurare che il software sia utile e intuitivo per gli utenti

Attività Principali della Verifica e Validazione

  • Test di unità
  • Test di integrazione
  • Test di sistema
  • Test di accettazione

3. Esempio di Analisi di Flusso dei Dati

Nel seguente esempio, analizziamo il comportamento delle variabili all'interno della funzione swap() e come vengono definite (D), usate (U) o assegnate (A).

void swap(int *x, int *y) {{
  int z;        // (1)
  *y = z;       // (2)
  *y = *x;      // (3)
  *x = z;       // (4)
}}

Analisi delle variabili:

  • x:
    - D → passaggio parametro
    - U → (3)
    - D → (4)
  • y:
    - D → passaggio parametro
    - D → (2)
    - D → (3)
  • z:
    - A → (1)
    - U → (2)
    - U → (4)
